# Domain Model

## Core Entities

### Request
Created and validated in Module 01. Consumed by Module 02.

### RequestServiceNeed
A specific service need inside a request.

Examples:

- Bathing/care
- Diaper change
- Injection
- Physiotherapy

### ServiceProvider
Abstract provider concept covering:

- Independent Provider
- Company Provider
- Company

### IndependentProvider
A provider who works independently.

### Company
A provider organization.

### CompanyProvider
A provider affiliated with a company.

### ProviderServiceProfile
Service-specific profile for a provider.

Fields:

- service type
- price
- experience
- documents
- active status
- qualification status

### ServiceCoverage
Defines where provider offers service.

Supports city, area, radius, and future polygon.

### ProviderAvailability
Provider working calendar.

### EligibilityEvaluation
Stores the result of eligibility checks.

### MatchRound
A matching execution for a request or service need.

### MatchCandidate
A candidate generated by Matching Engine.

Types:

- Independent Provider
- Company Provider
- Company
- Package by individual
- Package by company

### CandidateResponse
Provider response to matching request.

Actions:

- Accepted
- Rejected
- Withdrawn

### CandidateRankingScore
Stores scoring components and final ranking value.

### CandidatePresentationCard
Read model for customer-facing display.

### Recommendation
Future explainable recommendation object.

### CustomerSelection
Records selected candidate for a service need or package.

### NotificationEvent
Tracks notification sent through configurable channels.

### ManualInterventionLog
Audit log for support actions.

## Key Relationships

```text
Request 1..n RequestServiceNeed
RequestServiceNeed 1..n MatchCandidate
ServiceProvider 1..n ProviderServiceProfile
Company 1..n CompanyProvider
MatchCandidate 0..n CandidateResponse
MatchCandidate 0..1 CustomerSelection
MatchCandidate 1..1 CandidateRankingScore
```

## Provider Lifecycle

Applies to independent provider, company provider, and company:

```text
REGISTERED
PROFILE_INCOMPLETE
PENDING_REVIEW
PENDING_DOCUMENTS
APPROVED
ACTIVE
TEMPORARILY_UNAVAILABLE
SUSPENDED
UNDER_APPEAL
DEACTIVATED
TERMINATED
```

Only `ACTIVE` enters Matching in MVP.
