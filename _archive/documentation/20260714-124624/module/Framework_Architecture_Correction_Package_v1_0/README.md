# Framework Architecture Correction Package v1.0

This package corrects Modules 01–08 so they can be treated as a Generic Service Marketplace Framework rather than a senior-care-specific architecture.

## Purpose

The correction package defines the canonical cross-module rules for:

- Generic actor vocabulary
- Identity and access ownership
- Communication ownership
- Trust versus identity boundaries
- CES event naming
- CCS configuration naming
- Cross-module dependency rules
- Freeze gates for Modules 01–08

## Binding Rule

When a module document conflicts with this correction package, this correction package wins until the module is individually rewritten and re-frozen.
